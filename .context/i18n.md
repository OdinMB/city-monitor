# Internationalization (i18n)

## Stack

`i18next` + `react-i18next`. Initialized in `packages/web/src/i18n/index.ts`.

## Languages

| Code | Language | Status |
|---|---|---|
| `de` | German | Full coverage |
| `en` | English | Full coverage |
| `tr` | Turkish | Full coverage |
| `ar` | Arabic | Full coverage |

## Translation Files

JSON files in `packages/web/src/i18n/`:
- `de.json`, `en.json`, `tr.json`, `ar.json`

## Key Structure

```
app.title, app.noscript
topbar.language, topbar.theme.light, topbar.theme.dark
panel.news.{title, briefing, headlines, empty, generatedAgo, headlines_count, all, retry}
panel.weather.{title, humidity, wind, feelsLike, hourly, daily, high, low, alerts, validUntil}
panel.transit.{title, empty, affectedStops}
panel.events.{title, empty, free}
panel.safety.{title, empty}
panel.map.title
category.{transit, politics, culture, crime, weather, economy, sports, local, music, art, theater, food, market, sport, community, other}
time.{justNow, minutesAgo, hoursAgo, daysAgo}
footer.{sourceCode, basedOn, license}
```

## Language Detection

Priority: `localStorage` → `navigator.language` → fallback to `en`.

## Usage in Components

```typescript
import { useTranslation } from 'react-i18next';
const { t, i18n } = useTranslation();
t('panel.news.title')  // → "News" or "Nachrichten"
i18n.changeLanguage('de')
```

## Language Switcher

TopBar renders DE/EN/TR/AR buttons. Active language highlighted. Stored in localStorage by i18next.

## Testing

`test-setup.ts` initializes i18n with English translations so test assertions match English strings.

## Adding a Language

1. Create `packages/web/src/i18n/{code}.json` with all keys
2. Import and register in `packages/web/src/i18n/index.ts`
3. Add to `LANGUAGES` array in `TopBar.tsx`
